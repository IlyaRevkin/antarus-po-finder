# Antarus ПО Finder — Руководство пользователя

## Что это такое

Antarus ПО Finder — десктопное приложение для хранения, поиска и управления прошивками оборудования.
Наладчик вводит название шкафа → приложение находит подходящую прошивку → показывает версию, инструкции, параметры.

---

## Запуск

Запустите `Antarus_PO_Finder.exe`. При первом запуске автоматически создаётся база данных в папке
`C:\Users\ИМЯ_ПОЛЬЗОВАТЕЛЯ\AppData\Local\FirmwareFinder\firmware_finder.db`.

---

## Роли

При запуске приложение спрашивает роль. Роль определяет, какие разделы доступны:

| Роль | Поиск | Загрузка | Шаблоны | Настройки |
|------|:-----:|:--------:|:-------:|:---------:|
| Наладчик | ✓ | — | — | — |
| Нал-Администратор | ✓ | — | ✓ | — |
| Программист | ✓ | ✓ | ✓ | — |
| Администратор | ✓ | ✓ | ✓ | ✓ |

Сменить роль можно кнопкой **«Сменить роль»** в боковой панели.
Пароли на роли «Программист» и «Администратор» задаются в **Настройки → Общие → Пароли**.
По умолчанию пароль администратора: `12345`.

## Раздел: Поиск

Главный раздел для наладчиков.

### Как искать

1. Введите название шкафа в поле поиска, например: `НГР КПЧ SMH5` или `ПЖ KINCO`
2. Нажмите **«Найти»** или Enter
3. Приложение показывает карточки совпавших правил с версиями прошивок

### Карточка прошивки

Каждая карточка содержит:
- **Название правила** и тип оборудования
- **Актуальная версия** (самая новая)
- Кнопки действий:
  - **Открыть** — открыть файл прошивки из локального кеша
  - **Синхронизировать** — скопировать прошивку с сетевого диска (появляется если локальной копии нет)
  - **Обновить** — загрузить новую версию с диска (появляется если локальная копия устарела)
  - **Карта in/out** — открыть файл карты входов/выходов (если указан в правиле)
  - **Параметры** — открыть шаблоны параметров ПЧ/КПЧ/УПП
  - **Инструкции** — открыть инструкцию (если указана в правиле)
  - **Паспорт** — открыть паспорт оборудования (если указан)
  - **История** — просмотр всех версий с описаниями и changelog
  - **Копировать** — скопировать название прошивки в буфер обмена

### Кнопка «Копировать»

Копирует название в формате `ПРАВИЛО_ВЕРСИЯ`, например: `НГР_КПЧ_SMH5_3.42.260414`
Всё в верхнем регистре, пробелы заменены подчёркиваниями.

### История версий

Нажмите **«История»** на карточке. Откроется окно с:
- Таблицей всех версий (версия, дата, контроллер, краткое описание)
- Панелью внизу с полным описанием и changelog выбранной строки

### Быстрые приложения

Над поиском расположена панель быстрого запуска (SMLogix и др.).
Настраивается в **Настройки → Приложения**.

### Папка осмотра (протокол)

Панель над поиском позволяет:
- **Открыть** — открыть папку для протоколов осмотра
- **Выбрать** — выбрать другую папку
- **Сканировать** — запустить WIA-сканер (или Windows Scan как запасной вариант)
- **Фото** — поднять HTTP-сервер и показать QR-код для загрузки фото со смартфона

Фото сохраняются **напрямую** в выбранную папку осмотра (подпапки не создаются).

## Раздел: Загрузка

Доступен программистам и администраторам.

### Загрузка новой прошивки

1. Перетащите файл прошивки в зону **«Перетащите файл или папку сюда»** или кликните для выбора
2. Нажмите **«Определить автоматически»** — приложение попытается определить версию по имени файла
3. Заполните поля формы:
   - **Версия** — в формате `префикс.номер.ГГММДД`, например `3.42.260414`
   - **Контроллер** — тип контроллера (SMH5, KINCO и т.д.)
   - **Тип оборудования** — НГР, ПЖ, ТГР и т.д.
   - **Тип работы** — ПП, УПП, ПЧ, КПЧ
   - **Описание** — краткое описание версии
   - **Изменения** — список изменений
   - **Карта in/out** — путь к файлу карты (можно перетащить, нажать «×» чтобы очистить)
   - **Инструкция** — путь к файлу/папке инструкций (можно перетащить, нажать «×» чтобы очистить)
4. В левой части выберите **правила** (шкафы), для которых эта прошивка предназначена
5. Нажмите **«Загрузить прошивку»**

### Автоматическое создание правил

Если не выбрано ни одно правило, приложение создаёт правило автоматически:
- Ищет совпадение по типу оборудования **и** контроллеру одновременно
- Если совпадение не найдено — создаёт новое правило

Карта in/out и инструкция из формы загрузки автоматически сохраняются в правило
(если в правиле ещё не указаны).

### Имена файлов

При сохранении прошивки имя файла нормализуется:
- переводится в **ВЕРХНИЙ РЕГИСТР**
- пробелы заменяются **подчёркиваниями**

Например: `firmware ngr v3.42.exe` → `FIRMWARE_NGR_V3.42.EXE`

### Формат версии

Версия состоит из трёх частей через точку: `prefix.major.YYMMDD`
- `prefix` — числовой код типа оборудования (3 = КПЧ, 2 = ПЧ и т.д., задаётся в настройках)
- `major` — номер версии
- `YYMMDD` — дата (год-месяц-день), проставляется автоматически при потере фокуса

## Раздел: Шаблоны

Управление шаблонами параметров ПЧ, КПЧ, УПП.

- Можно добавлять/удалять/редактировать шаблоны
- Каждый шаблон привязан к одному или нескольким правилам
- Типы: `pch` (параметры ПЧ/КПЧ) и `upp` (параметры УПП)

---

## Раздел: Настройки

Доступен только администраторам.

### Вкладки настроек

**Общие:**
- Путь к сетевому диску с прошивками
- Пароли ролей
- Прочие параметры

**Правила:**
- Управление правилами соответствия (поиск → папка с прошивками)
- Каждое правило содержит: ключевые слова для поиска, папку, контроллер, тип,
  пути к картам/инструкциям/паспортам/параметрам
- Для полей путей (карта in/out, инструкции, паспорт и т.д.) кнопка **«×»** очищает значение

**Префиксы:**
- Соответствие типа оборудования → числовой префикс версии

**Приложения:**
- Быстрые приложения (SMLogix и др.)

**Типы:**
- Редактирование списков типов оборудования, контроллеров, работ

---

## Синхронизация с диском

Приложение каждые 5 минут проверяет наличие сетевого диска и синхронизирует локальный кеш.
Статус отображается в нижней строке состояния: `Диск: ✓ (42 файлов)` или `Диск: ✗ недоступен`.
